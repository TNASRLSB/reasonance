# Reasonance 文档

## 1. 简介

### 什么是 Reasonance
Reasonance 是一款为 vibecoder 设计的轻量级桌面 IDE —— 即与 AI 编程助手协作的开发者。它提供了一个简洁、专注的环境，集成了 LLM 终端、代码编辑器和文件管理功能。

### 系统要求
- Linux（推荐 KDE Plasma）、macOS 或 Windows
- 至少安装一个 LLM CLI 工具（Claude Code、Ollama 等）

### 安装
从 GitHub Releases 页面下载最新版本。在 Linux 上，安装 .deb 或 .AppImage 包。

## 2. 界面

### 布局
Reasonance 使用三面板布局：
- **左面板**：文件树，用于浏览项目
- **中间面板**：代码编辑器，带语法高亮
- **右面板**：LLM 终端，用于 AI 辅助编程

### 菜单栏
从菜单栏访问所有功能：
- **文件**：打开文件夹、管理文件、最近项目
- **编辑**：撤销、重做、剪贴板、搜索
- **视图**：主题、可读性、面板可见性
- **终端**：创建 LLM 终端、YOLO 模式
- **Git**：状态、提交、推送、拉取、日志
- **帮助**：文档、键盘快捷键

### 状态栏
底部状态栏显示：
- 应用名称和检测到的 LLM 数量
- 活动终端会话信息（上下文百分比、模型、重置计时器、消息数）
- 活动文件信息（名称、语言、编码）
- YOLO 模式指示器（激活时为红色条）

### 键盘快捷键
| 快捷键 | 操作 |
|--------|------|
| Ctrl+P | 快速文件搜索 |
| Ctrl+Shift+F | 在文件中搜索 |
| Ctrl+S | 保存文件 |
| Ctrl+, | 打开设置 |
| F1 | 打开文档 |

## 3. 文件管理

### 打开项目
使用**文件 > 打开文件夹**或在欢迎屏幕上点击"打开文件夹"。最近的项目列在快速访问列表中。

### 浏览文件
在文件树中点击文件打开它们。右键点击查看上下文菜单操作。使用 Ctrl+P 按名称快速搜索文件。

### 编辑文件
文件默认以只读模式打开。点击"只读"切换编辑模式。更改通过影子副本进行跟踪，用于差异检测。

### 搜索
- **Ctrl+P**：按名称搜索文件
- **Ctrl+Shift+F**：搜索文件内容（grep）

## 4. LLM 终端

### 启动 LLM
点击终端面板中的 **+** 按钮查看可用的 LLM。Reasonance 自动检测已安装的 CLI 工具（Claude Code、Ollama 等）。

### 多个实例
同时运行多个 LLM 会话。每个实例有自己的标签页。使用标签栏在实例之间切换。

### YOLO 模式
从工具栏或**终端 > YOLO 模式**启用 YOLO 模式。这会将 --dangerously-skip-permissions 标志传递给 Claude Code，允许它在没有确认提示的情况下运行。状态栏变为红色作为警告。

### 上下文跟踪
状态栏显示从 LLM 输出中解析的实时上下文窗口使用情况，包括：
- 会话使用百分比及可视化条
- 活动模型名称
- 剩余消息数
- 重置倒计时器

## 5. Git 集成

从 **Git** 菜单访问 Git 命令。命令在活动终端中运行：
- **状态**：显示工作树状态
- **提交**：开始提交（输入你的消息）
- **推送**：推送到远程
- **拉取**：从远程拉取
- **日志**：显示最近的提交历史

## 6. 设置

使用 **Ctrl+,** 或齿轮图标打开设置。

### 主题
在浅色、深色或系统（跟随操作系统偏好）之间选择。在 KDE/Wayland 上，系统模式使用原生检测，回退到深色。

### 语言
从 9 种语言中选择：English、Italiano、Deutsch、Espanol、Francais、Portugues、中文、Hindi、Al-Arabiya。阿拉伯语启用 RTL 布局。

### 字体与可读性
- 自定义字体系列和大小
- 增强可读性模式：更大的文本、增加的间距、优化的无障碍访问

### LLM 配置
LLM 在首次启动时自动检测。高级设置可通过 TOML 配置文件手动配置。

## 7. 故障排除

### LLM 未检测到
- 确保 LLM CLI 工具已安装并在你的 PATH 中
- 尝试**终端 > 检测 LLM** 重新扫描
- 检查配置文件进行手动配置

### Linux 上渲染模糊
Reasonance 包含了针对 KDE/Wayland (WebKitGTK) 分数缩放的修复。如果渲染仍然模糊，请检查你的显示缩放设置。

### 主题不切换
如果主题不响应系统变化，尝试在设置中明确设置为浅色或深色，然后切换回系统。

### 常见问题
**问：我可以同时使用多个 LLM 吗？**
答：可以，每个 LLM 有自己的标签页。点击 + 添加更多实例。

**问：如何配置自定义 LLM？**
答：编辑 ~/.config/reasonance/config.toml 中的 TOML 配置文件。

**问：YOLO 模式适用于所有 LLM 吗？**
答：YOLO 模式目前针对 Claude Code 进行了优化。其他 LLM 可能有不同的确认机制。
