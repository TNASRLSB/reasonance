# وثائق Reasonance

## 1. مقدمة

### ما هو Reasonance
Reasonance هو بيئة تطوير متكاملة خفيفة الوزن مصممة للمطورين الذين يعملون مع مساعدي البرمجة المدعومين بالذكاء الاصطناعي. يوفر بيئة نظيفة ومركزة مع طرفيات LLM مدمجة ومحرر أكواد وإدارة ملفات.

### متطلبات النظام
- Linux (يُنصح بـ KDE Plasma) أو macOS أو Windows
- أداة CLI واحدة على الأقل لـ LLM مثبتة (Claude Code، Ollama، إلخ)

### التثبيت
قم بتنزيل أحدث إصدار من صفحة GitHub Releases. على Linux، قم بتثبيت حزمة .deb أو .AppImage.

## 2. الواجهة

### التخطيط
يستخدم Reasonance تخطيطًا من ثلاث لوحات:
- **اللوحة اليسرى**: شجرة الملفات للتنقل في مشروعك
- **اللوحة الوسطى**: محرر الأكواد مع تمييز بناء الجملة
- **اللوحة اليمنى**: طرفية LLM للبرمجة بمساعدة الذكاء الاصطناعي

### شريط القوائم
الوصول إلى جميع الميزات من شريط القوائم:
- **ملف**: فتح المجلدات، إدارة الملفات، المشاريع الأخيرة
- **تحرير**: تراجع، إعادة، الحافظة، بحث
- **عرض**: السمة، سهولة القراءة، رؤية اللوحات
- **الطرفية**: إنشاء طرفيات LLM، وضع YOLO
- **Git**: الحالة، الإيداع، الدفع، السحب، السجل
- **المساعدة**: الوثائق، اختصارات لوحة المفاتيح

### شريط الحالة
يعرض شريط الحالة السفلي:
- اسم التطبيق وعدد LLM المكتشفة
- معلومات جلسة الطرفية النشطة (السياق %، النموذج، مؤقت إعادة التعيين، الرسائل)
- معلومات الملف النشط (الاسم، اللغة، الترميز)
- مؤشر وضع YOLO (شريط أحمر عند التفعيل)

### اختصارات لوحة المفاتيح
| الاختصار | الإجراء |
|----------|---------|
| Ctrl+P | بحث سريع عن الملفات |
| Ctrl+Shift+F | البحث في الملفات |
| Ctrl+S | حفظ الملف |
| Ctrl+, | فتح الإعدادات |
| F1 | فتح الوثائق |

## 3. إدارة الملفات

### فتح مشروع
استخدم **ملف > فتح مجلد** أو انقر على "فتح مجلد" في شاشة الترحيب. المشاريع الأخيرة مدرجة للوصول السريع.

### التنقل بين الملفات
انقر على الملفات في شجرة الملفات لفتحها. انقر بزر الماوس الأيمن لإجراءات القائمة السياقية. استخدم Ctrl+P للبحث السريع عن ملف بالاسم.

### تحرير الملفات
تفتح الملفات في وضع القراءة فقط بشكل افتراضي. انقر على "قراءة فقط" للتبديل إلى وضع التحرير. يتم تتبع التغييرات بنسخ ظلية لاكتشاف الفروقات.

### البحث
- **Ctrl+P**: البحث عن الملفات بالاسم
- **Ctrl+Shift+F**: البحث في محتوى الملفات (grep)

## 4. طرفية LLM

### تشغيل LLM
انقر على زر **+** في لوحة الطرفية لرؤية LLM المتاحة. يكتشف Reasonance تلقائيًا أدوات CLI المثبتة (Claude Code، Ollama، إلخ).

### مثيلات متعددة
قم بتشغيل جلسات LLM متعددة في وقت واحد. لكل مثيل علامة تبويب خاصة به. التبديل بين المثيلات باستخدام شريط علامات التبويب.

### وضع YOLO
قم بتفعيل وضع YOLO من شريط الأدوات أو **الطرفية > وضع YOLO**. يمرر هذا علامة --dangerously-skip-permissions إلى Claude Code، مما يسمح له بالتشغيل دون مطالبات التأكيد. يتحول شريط الحالة إلى اللون الأحمر كتحذير.

### تتبع السياق
يعرض شريط الحالة استخدام نافذة السياق في الوقت الفعلي، المحلل من مخرجات LLM، بما في ذلك:
- نسبة استخدام الجلسة مع شريط مرئي
- اسم النموذج النشط
- الرسائل المتبقية
- مؤقت العد التنازلي لإعادة التعيين

## 5. تكامل Git

الوصول إلى أوامر Git من قائمة **Git**. تعمل الأوامر في الطرفية النشطة:
- **الحالة**: عرض حالة شجرة العمل
- **الإيداع**: بدء إيداع (اكتب رسالتك)
- **الدفع**: الدفع إلى المستودع البعيد
- **السحب**: السحب من المستودع البعيد
- **السجل**: عرض سجل الإيداعات الأخيرة

## 6. الإعدادات

افتح الإعدادات بـ **Ctrl+,** أو أيقونة الترس.

### السمة
اختر بين فاتح أو داكن أو النظام (يتبع تفضيل نظام التشغيل). على KDE/Wayland، يستخدم وضع النظام الاكتشاف الأصلي مع الرجوع إلى الداكن.

### اللغة
اختر من بين 9 لغات: English، Italiano، Deutsch، Espanol، Francais، Portugues، 中文، हिन्दी، العربية. العربية تفعّل تخطيط RTL.

### الخط وسهولة القراءة
- عائلة وحجم خط مخصصان
- وضع القراءة المحسّن: نص أكبر، تباعد أكثر، محسّن لإمكانية الوصول

### تكوين LLM
يتم اكتشاف LLM تلقائيًا عند التشغيل الأول. التكوين اليدوي عبر ملف تكوين TOML للإعدادات المتقدمة.

## 7. استكشاف الأخطاء وإصلاحها

### لم يتم اكتشاف LLM
- تأكد من أن أداة CLI الخاصة بـ LLM مثبتة وفي PATH الخاص بك
- جرّب **الطرفية > اكتشاف LLM** لإعادة المسح
- تحقق من ملف التكوين للتكوين اليدوي

### العرض ضبابي على Linux
يتضمن Reasonance إصلاحًا للتحجيم الكسري على KDE/Wayland (WebKitGTK). إذا كان العرض لا يزال ضبابيًا، تحقق من إعدادات تحجيم العرض.

### السمة لا تتغير
إذا لم تستجب السمة لتغييرات النظام، جرّب تعيينها صراحةً إلى فاتح أو داكن في الإعدادات، ثم العودة إلى النظام.

### الأسئلة الشائعة
**س: هل يمكنني استخدام عدة LLM في وقت واحد؟**
ج: نعم، لكل LLM علامة تبويب خاصة به. انقر على + لإضافة مزيد من المثيلات.

**س: كيف أقوم بتكوين LLM مخصص؟**
ج: قم بتحرير ملف تكوين TOML في ~/.config/reasonance/config.toml

**س: هل يعمل وضع YOLO مع جميع LLM؟**
ج: وضع YOLO محسّن حاليًا لـ Claude Code. قد تمتلك LLM الأخرى آليات تأكيد مختلفة.
