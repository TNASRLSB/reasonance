# Reasonance प्रलेखन

## 1. परिचय

### Reasonance क्या है
Reasonance वाइबकोडर्स के लिए डिज़ाइन किया गया एक हल्का डेस्कटॉप IDE है — ऐसे डेवलपर्स जो AI-संचालित कोडिंग सहायकों के साथ काम करते हैं। यह एकीकृत LLM टर्मिनल, एक कोड एडिटर और फ़ाइल प्रबंधन के साथ एक स्वच्छ, केंद्रित वातावरण प्रदान करता है।

### सिस्टम आवश्यकताएँ
- Linux (KDE Plasma अनुशंसित), macOS, या Windows
- कम से कम एक LLM CLI टूल इंस्टॉल होना चाहिए (Claude Code, Ollama, आदि)

### इंस्टॉलेशन
GitHub Releases पेज से नवीनतम संस्करण डाउनलोड करें। Linux पर, .deb या .AppImage पैकेज इंस्टॉल करें।

## 2. इंटरफ़ेस

### लेआउट
Reasonance तीन-पैनल लेआउट का उपयोग करता है:
- **बायाँ पैनल**: प्रोजेक्ट नेविगेशन के लिए फ़ाइल ट्री
- **मध्य पैनल**: सिंटैक्स हाइलाइटिंग के साथ कोड एडिटर
- **दायाँ पैनल**: AI-संचालित कोडिंग के लिए LLM टर्मिनल

### मेनू बार
मेनू बार से सभी सुविधाओं तक पहुँचें:
- **फ़ाइल**: फ़ोल्डर खोलें, फ़ाइलें प्रबंधित करें, हाल के प्रोजेक्ट
- **संपादन**: पूर्ववत करें, फिर से करें, क्लिपबोर्ड, खोज
- **दृश्य**: थीम, पठनीयता, पैनल दृश्यता
- **टर्मिनल**: LLM टर्मिनल बनाएँ, YOLO मोड
- **Git**: स्थिति, कमिट, पुश, पुल, लॉग
- **सहायता**: प्रलेखन, कीबोर्ड शॉर्टकट

### स्टेटस बार
निचला स्टेटस बार दिखाता है:
- ऐप का नाम और पहचाने गए LLM की संख्या
- सक्रिय टर्मिनल सत्र जानकारी (संदर्भ %, मॉडल, रीसेट टाइमर, संदेश)
- सक्रिय फ़ाइल जानकारी (नाम, भाषा, एन्कोडिंग)
- YOLO मोड संकेतक (सक्रिय होने पर लाल बार)

### कीबोर्ड शॉर्टकट
| शॉर्टकट | क्रिया |
|----------|--------|
| Ctrl+P | त्वरित फ़ाइल खोज |
| Ctrl+Shift+F | फ़ाइलों में खोजें |
| Ctrl+S | फ़ाइल सहेजें |
| Ctrl+, | सेटिंग्स खोलें |
| F1 | प्रलेखन खोलें |

## 3. फ़ाइल प्रबंधन

### प्रोजेक्ट खोलना
**फ़ाइल > फ़ोल्डर खोलें** का उपयोग करें या स्वागत स्क्रीन पर "फ़ोल्डर खोलें" पर क्लिक करें। हाल के प्रोजेक्ट त्वरित पहुँच के लिए सूचीबद्ध हैं।

### फ़ाइलें नेविगेट करना
फ़ाइल ट्री में फ़ाइलों पर क्लिक करके उन्हें खोलें। संदर्भ मेनू क्रियाओं के लिए राइट-क्लिक करें। नाम से त्वरित फ़ाइल खोज के लिए Ctrl+P का उपयोग करें।

### फ़ाइलें संपादित करना
फ़ाइलें डिफ़ॉल्ट रूप से केवल-पठन मोड में खुलती हैं। संपादन मोड टॉगल करने के लिए "केवल-पठन" पर क्लिक करें। परिवर्तनों को अंतर पहचान के लिए छाया प्रतियों के साथ ट्रैक किया जाता है।

### खोज
- **Ctrl+P**: नाम से फ़ाइलें खोजें
- **Ctrl+Shift+F**: फ़ाइल सामग्री में खोजें (grep)

## 4. LLM टर्मिनल

### LLM शुरू करना
उपलब्ध LLM देखने के लिए टर्मिनल पैनल में **+** बटन पर क्लिक करें। Reasonance स्वचालित रूप से इंस्टॉल किए गए CLI टूल्स (Claude Code, Ollama, आदि) का पता लगाता है।

### एकाधिक इंस्टेंस
एक साथ कई LLM सत्र चलाएँ। प्रत्येक इंस्टेंस का अपना टैब होता है। टैब बार का उपयोग करके इंस्टेंस के बीच स्विच करें।

### YOLO मोड
टूलबार या **टर्मिनल > YOLO मोड** से YOLO मोड सक्षम करें। यह Claude Code को --dangerously-skip-permissions फ़्लैग पास करता है, जिससे यह पुष्टि संकेतों के बिना चल सकता है। स्टेटस बार चेतावनी के रूप में लाल हो जाता है।

### संदर्भ ट्रैकिंग
स्टेटस बार LLM आउटपुट से पार्स किए गए रीयल-टाइम संदर्भ विंडो उपयोग को प्रदर्शित करता है, जिसमें शामिल हैं:
- विज़ुअल बार के साथ सत्र उपयोग प्रतिशत
- सक्रिय मॉडल का नाम
- शेष संदेश
- रीसेट काउंटडाउन टाइमर

## 5. Git एकीकरण

**Git** मेनू से Git कमांड एक्सेस करें। कमांड सक्रिय टर्मिनल में चलते हैं:
- **स्थिति**: कार्य ट्री स्थिति दिखाएँ
- **कमिट**: एक कमिट शुरू करें (अपना संदेश टाइप करें)
- **पुश**: रिमोट पर पुश करें
- **पुल**: रिमोट से पुल करें
- **लॉग**: हाल का कमिट इतिहास दिखाएँ

## 6. सेटिंग्स

**Ctrl+,** या गियर आइकन से सेटिंग्स खोलें।

### थीम
लाइट, डार्क, या सिस्टम (OS प्राथमिकता का अनुसरण करता है) में से चुनें। KDE/Wayland पर, सिस्टम मोड डार्क पर फ़ॉलबैक के साथ नेटिव डिटेक्शन का उपयोग करता है।

### भाषा
9 भाषाओं में से चुनें: English, Italiano, Deutsch, Espanol, Francais, Portugues, 中文, हिन्दी, العربية। अरबी RTL लेआउट सक्षम करती है।

### फ़ॉन्ट और पठनीयता
- कस्टम फ़ॉन्ट फ़ैमिली और आकार
- उन्नत पठनीयता मोड: बड़ा टेक्स्ट, बढ़ी हुई स्पेसिंग, एक्सेसिबिलिटी के लिए अनुकूलित

### LLM कॉन्फ़िगरेशन
पहली बार लॉन्च पर LLM स्वचालित रूप से पहचाने जाते हैं। उन्नत सेटअप के लिए TOML कॉन्फ़िग फ़ाइल के माध्यम से मैन्युअल कॉन्फ़िगरेशन।

## 7. समस्या निवारण

### LLM नहीं पहचाने गए
- सुनिश्चित करें कि LLM CLI टूल इंस्टॉल है और आपके PATH में है
- पुनः स्कैन करने के लिए **टर्मिनल > LLM पहचानें** प्रयास करें
- मैन्युअल कॉन्फ़िगरेशन के लिए कॉन्फ़िग फ़ाइल जाँचें

### Linux पर धुंधला रेंडरिंग
Reasonance में KDE/Wayland (WebKitGTK) पर फ़्रैक्शनल स्केलिंग के लिए एक फ़िक्स शामिल है। यदि रेंडरिंग अभी भी धुंधला है, तो अपनी डिस्प्ले स्केलिंग सेटिंग्स जाँचें।

### थीम नहीं बदल रही
यदि थीम सिस्टम परिवर्तनों पर प्रतिक्रिया नहीं करती, तो सेटिंग्स में इसे स्पष्ट रूप से लाइट या डार्क पर सेट करने का प्रयास करें, फिर सिस्टम पर वापस जाएँ।

### अक्सर पूछे जाने वाले प्रश्न
**प्र: क्या मैं एक साथ कई LLM का उपयोग कर सकता हूँ?**
उ: हाँ, प्रत्येक LLM का अपना टैब होता है। अधिक इंस्टेंस जोड़ने के लिए + पर क्लिक करें।

**प्र: मैं कस्टम LLM कैसे कॉन्फ़िगर करूँ?**
उ: ~/.config/reasonance/config.toml पर TOML कॉन्फ़िग फ़ाइल संपादित करें।

**प्र: क्या YOLO मोड सभी LLM के साथ काम करता है?**
उ: YOLO मोड वर्तमान में Claude Code के लिए अनुकूलित है। अन्य LLM में अलग पुष्टि तंत्र हो सकते हैं।
